APIキー取得方法（Google Maps）

Google アカウント作成 → プロジェクト作成 → API 有効化 → API キー発行 の順で進めます。

  1. Google アカウントでログイン／作成

    【Google アカウント】をお持ちでない場合は、先に登録（無料）をお願いします。

    Google アカウントを作成

  2. Google Cloud コンソールを開く

    ブラウザで以下の URL を開き、先ほどの Google アカウントでサインインします。

    Google Cloud Console

  3. プロジェクトを作成する

    • 画面上部メニューの「プロジェクトを選択」をクリック
    • 右上の「＋ 新しいプロジェクト」をクリック
    • プロジェクト名（例：「Vaulta」）を入力し、「作成」をクリック

    数秒後、自動でそのプロジェクトが選択されます。

  4. 必要な API を有効化する

    • 左メニュー「API とサービス」→「ライブラリ」
    • 検索ボックスに Maps JavaScript API と入力 → 該当 API を選択
    • 「有効にする」をクリック
    • 同様に Directions API を検索して開き、「有効にする」をクリック

    Vaulta ではルート検索に Directions API を使用します。必ず有効化してください。

  5. API キーを作成する

    • 左メニュー「API とサービス」→「認証情報」
    • 画面上部の「認証情報を作成」→「API キー」
    • 「API キーを作成しました」ポップアップに表示されたキー（例：AIzaSyA-xxxxxxxxxxxxxxxxxxxx）をコピーし、Vaulta の設定欄に貼り付けてください。

  6. API キーの制限を設定する（推奨）

    第三者による不正利用を防ぐため、使用できる API を限定することをおすすめします。

    • 「API とサービス」→「認証情報」から、先ほど作成した API キーをクリック
    • 「API の制限」セクションで「キーを制限」を選択
    • API の一覧から、少なくとも次の API にチェックを入れます：
      • Directions API（必須：Vaulta のルート検索で使用）
    • 画面下部の「保存」をクリック

    制限を設定したあとにアプリが動かない場合は、この画面で Directions API が選択されているかを再度確認してください。

参考 URL

  • Google Cloud コンソール（プロジェクト作成／API 有効化／認証情報作成）
    https://console.cloud.google.com/
  • Google Maps JavaScript API キー取得ガイド（公式ドキュメント・日本語）
    https://developers.google.com/maps/documentation/javascript/get-api-key?hl=ja
補足：API キーは第三者に知られないよう厳重に管理してください。誤って公開してしまった場合は、認証情報画面からキーを「再生成」し、アプリ側の設定も更新してください。
