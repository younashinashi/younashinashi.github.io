Vaulta プライバシーポリシー

最終更新日：

1. 適用範囲

本プライバシーポリシー（以下「本ポリシー」といいます）は、スマートフォンアプリ「Vaulta」（以下「本アプリ」といいます）の利用に伴い、個人開発者 younashinashi（以下「開発者」といいます）が取り扱う情報およびその管理方針を定めるものです。本アプリをインストールし利用するすべての方（以下「ユーザー」といいます）は、本ポリシーに同意したものとみなされます。

2. 運営者情報

本アプリは個人開発者 younashinashi が開発・運営しています。ご連絡は younashinashi@gmail.com までお願いいたします。

3. 開発者が取得する情報について

開発者は、本アプリを通じて次の情報をインターネット経由で取得・収集しません。

  • 氏名・住所・電話番号・メールアドレスなどの個人情報
  • OS の種類、端末識別子、広告IDなどの端末情報
  • ログイン日時や機能利用履歴などの利用状況データ
  • クッキー、広告識別子その他類似技術による識別情報
  • 位置情報を含む各種センサー情報

本アプリの設計上、開発者サーバーへのユーザー情報送信は行っておらず、開発者がユーザー個人を識別できる情報を取得することはありません。

4. アプリ内で扱うデータとローカル保存

本アプリでは、ユーザーによる入力に基づき、訪問予定、利用者名・住所、体調メモなど、個人情報を含む可能性のあるデータを端末内で取り扱います。

  • これらのデータは、端末内のローカルストレージ（暗号化された Hive 領域等）に保存され、開発者サーバーや第三者サーバーには送信されません。
  • ユーザーが本アプリをアンインストールした場合、アプリが管理するデータは端末内から削除されます。
  • ただし、OS の自動バックアップ機能やユーザーご自身のバックアップ設定により、端末またはクラウドに複製・キャッシュが残存する可能性があり、開発者はそれらを管理・削除することはできません。

5. 端末機能・権限の利用

本アプリは、訪問予定の整理および現場メモ管理のために、主として次の端末機能を利用します。

  • インターネット接続：Google マップ API による移動時間・経路の取得などに利用します。
  • 通知機能：必要に応じて端末の通知領域に情報を表示する場合がありますが、氏名や住所などの詳細情報が直接表示されないよう配慮します。

現時点で本アプリは、カメラ、マイク、連絡先、写真ライブラリなどへのアクセス権限を要求しません。OS のバージョンアップや機能追加により権限が追加される場合は、本ポリシーを改定のうえ周知します。

6. 個人情報の利用目的

開発者は、第3項のとおりインターネット経由で個人情報を取得しないため、広告配信や行動解析等の目的でユーザー情報を利用することはありません。

端末内に保存されるデータは、ユーザー自身が本アプリ上で以下の目的で利用します。

  • 訪問予定・移動時間・空き時間の一覧表示
  • 利用者ごとの当日メモ・体調等の現場メモ記録および振り返り
  • スケジュールの可視化や自己の業務整理

これらのデータの内容および利用方法は、ユーザー自身の責任において管理されるものとします。

7. 個人情報の保存期間

開発者はユーザー個人を識別できる情報をサーバー側で保存しません。

  • 端末内に保存されたデータは、ユーザーが本アプリから削除するか、本アプリをアンインストールするまで端末内に保持されます。
  • アンインストール後も、OS やクラウドバックアップ等により複製が残存している場合がありますが、その管理・削除はユーザーと当該サービス提供者に委ねられ、開発者は関与できません。

8. 第三者提供

開発者は、ユーザー情報を外部委託先・提携先を含む第三者に提供・共有しません。

ユーザーが自らの判断でスクリーンショットや画面録画、テキストのコピーなどを行い、第三者に送信・共有した場合、それはユーザー自身の責任によるものであり、本ポリシーの対象外となります。

9. 外部サービス・SDK

本アプリは、以下の方針で外部サービスおよび SDK を利用します。

  • 広告 SDK や Firebase、Google Analytics 等の解析・トラッキング目的の SDK は使用していません。
  • 地図表示および移動時間取得のため、Google Maps Platform（API）を利用します。

Google Maps Platform の利用にあたり、住所や座標、経路情報などが Google に送信される場合があります。これらの情報の取り扱いは Google が定める利用規約およびプライバシーポリシーに従います。

  • ユーザーが自身で Google Cloud プロジェクトを作成し API キーを取得・設定する方式の場合、その契約・課金・利用制限等はすべてユーザーの責任において管理されます。
  • Google によるデータの利用・保管・第三者提供等について、開発者は関与せず、一切の責任を負いません。

10. セキュリティ対策

本アプリは、訪問介護の現場で扱われる情報の性質に配慮し、以下のようなセキュリティ対策を講じています。

  • ローカル保存のみ：クラウド同期・外部送信機能を設けず、端末内で完結する設計としています。
  • 暗号化：バックアップデータ等は AES-256-GCM などの方式で暗号化し、復号にはユーザーのパスフレーズから PBKDF2-HMAC-SHA256＋ソルトで導出した鍵を用います。
  • 改ざん検知：GCM の認証タグを利用することで、データ改ざんを検知します。
  • ランダムIV：暗号化時に毎回ランダムな初期化ベクトル（IV）を使用します。
  • PIN・生体認証：アプリ起動時ロックに 4桁 PIN コードや端末の生体認証機能を利用し、のぞき見リスクを低減します。
  • スクリーンショット・画面録画・タスクサムネイル対策：可能な範囲でスクリーンショット、画面録画、タスク一覧画面のサムネイル表示を制限・マスキングすることで、画面上の個人情報の撮影・共有リスクを抑制します。
  • 通知の匿名化：通知内には、氏名や住所などの詳細情報を直接表示せず、ID 等の最小限の情報のみを表示するよう配慮します。
  • コピー制御：UI 設計上、氏名・住所等が OS のクリップボードに長時間残存しないよう配慮しています。
  • 電話はダイヤラのみ：電話番号のタップ時には標準の電話アプリのダイヤラ画面のみを開き、自動発信や連絡先への自動保存は行いません。
  • ログ衛生：クラッシュログやデバッグログに、氏名・住所等の個人を特定し得る情報が含まれないよう配慮します。

これらの対策は情報漏えいリスクを低減するための合理的な措置であり、すべてのリスクを完全に排除するものではありません。OS の仕様、他アプリの挙動、端末の紛失・盗難、ユーザー自身のパスコード管理など、開発者の管理が及ばない要因に起因する漏えい・改ざん・消失について、開発者は一切の責任を負いません。

11. ユーザーの権利と管理

ユーザーは、本アプリに保存したデータを自身の操作により編集・削除できます。

  • 本アプリに関するすべてのローカルデータを削除したい場合は、端末から本アプリをアンインストールしてください。
  • OS やクラウドバックアップに保存されたデータの削除・管理方法については、各サービス提供者の案内をご確認ください。

12. プライバシーポリシーの変更

開発者は、法令の改正やサービス内容の変更、セキュリティ対策の強化等に応じて本ポリシーを改定することがあります。変更内容と施行日は、開発者のホームページまたは本アプリ内のお知らせ画面等で周知します。

ユーザーが改定後も本アプリを利用した場合、改定後の本ポリシーに同意したものとみなされます。

13. 準拠法・裁判管轄

本ポリシーは日本法を準拠法とし、本アプリまたは本ポリシーに関して紛争が生じた場合、開発者の所在地を管轄する日本の裁判所を第一審の専属的合意管轄裁判所とします。
